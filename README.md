# Kopern

**AI Agent Builder, Orchestrator & Grader** — Build custom AI agents, validate them through deterministic grading, optimize them in a 6-mode Optimization Lab, orchestrate multi-agent teams, and expose them as MCP servers for Claude Code, Cursor, or any compatible client. With Stripe billing, GitHub integration, and real-time observability.

Built with Next.js 16, React 19, Firebase, Stripe, and a multi-provider LLM streaming engine. Powered by [pi-mono](https://github.com/nicholasgasior/pi-mono), a 7-package TypeScript monorepo for stateful agent runtimes.

> **MVP Beta** — Kopern is in active development. Bug reports, feedback, and contributions are welcome!

## Features

### Core
- **Agent Builder** — Configure agents with system prompts, skills, custom tools, extensions, and branding
- **AI Agent Wizard** — Describe your agent in plain text, get a fully configured agent generated by AI
- **Multi-Model** — Anthropic, OpenAI, Google Gemini, Ollama (local) via a unified streaming interface
- **Extended Thinking** — 6 levels (off, minimal, low, medium, high, xhigh) for supported providers
- **Playground** — Live chat with SSE streaming, tool call visualization, and live metrics

### Grading & Optimization
- **Deterministic Grading** — 6 criteria: output match, schema validation, tool usage, safety check, custom script, LLM judge. Results persisted to Firestore
- **Optimization Lab** — 6 self-improving modes:
  - **AutoTune** — iterative prompt refinement via hill-climbing
  - **AutoFix** — one-click diagnosis and patch for failed test cases
  - **Stress Lab** — automated red team (prompt injection, jailbreaks, hallucination traps)
  - **Tournament** — A/B model arena to find the best config
  - **Distillation** — same quality, fraction of the cost
  - **Evolution** — parallel search across prompt × model × config space

### Orchestration
- **Agent Teams** — Parallel, sequential, or conditional multi-agent execution with real SSE
- **Pipelines** — Multi-step workflows with input mapping and per-step tool calling
- **Sub-agent Delegation** — Agents delegate subtasks to specialized sub-agents

### Deployment & Integration
- **MCP Protocol** — Real MCP Streamable HTTP server (`/api/mcp/server`) — native integration with Claude Code, Cursor, and any MCP client
- **MCP Endpoints** — Legacy JSON-RPC API (`/api/mcp`) with API key auth, rate limiting, and usage tracking
- **Connectors** — Deploy agents beyond the dashboard:
  - **Embeddable Widget** — Drop-in `<script>` tag for any website (Shadow DOM, SSE, markdown, mobile-responsive)
  - **Webhooks** — Inbound (sync JSON) + Outbound (fire-and-forget) with HMAC-SHA256 verification and anti-loop protection
  - **Slack Bot** — OAuth install, @mention/DM, thread context, async processing
- **Workflow Automation** — Native integration with n8n, Zapier, and Make via HTTP Request nodes:
  - **Inbound**: Use the platform's HTTP Request / Webhook node to call `POST /api/webhook/{agentId}?key=kpn_xxx`
  - **Outbound**: Configure outbound webhook URLs pointing to n8n/Zapier/Make catch hooks to trigger downstream workflows
  - **Anti-loop protection**: Inbound webhooks never fire outbound webhooks (prevents infinite loops between Kopern and external automation platforms)
- **GitHub Integration** — Connect repos via OAuth. Agents read files, search code, and get codebase context injected
- **Extensions** — 30+ event hooks with blocking support, accordion event selector UI

### Billing & Security
- **Stripe Billing** — Subscriptions (Pro $79/mo, Enterprise $499/mo), usage-based billing via Billing Meters, 9 webhook events, customer portal
- **Plan Enforcement** — `checkPlanLimits()` on all API routes (tokens, agents, grading, teams, pipelines, MCP, GitHub, models, autoresearch, version history)
- **Model Restriction** — Starter plan limited to Sonnet + Haiku; Pro+ unlocks all models

### Admin
- **Bug Fixer Agent** — Autonomous dev agent triggered by user bug reports: analyzes codebase, creates fix branch, opens PR, sends thank-you email
- **Bug Tracker** — Admin-only dashboard with KPI stats, status filters, expandable bug cards, notes timeline, and manual agent trigger
- **Builtin Tools** — `github_write` (all users) + `bug_management` (admin-only) with UI filtering via `NEXT_PUBLIC_ADMIN_UID`

### Platform
- **Sessions & Observability** — Server-side session tracking, conversation timelines, JSON export
- **Internationalization** — Full English/French with `[locale]` route segments, middleware detection, JSON dictionaries
- **Version Control** — Snapshot agents, track grading history per version
- **Authentication** — Firebase Auth (Google, GitHub with `repo` scope + credential linking, Email/Password)
- **Landing Page** — Hero agent creator, sticky navbar with accent glow, full documentation with search
- **Pricing Page** — 4 tiers with feature comparison table
- **Examples Gallery** — 15+ agent templates with one-click "Use this Agent" deployment
- **Documentation** — In-app docs (EN/FR) with full extension event reference, MCP tutorial, TOC, save as markdown
- **Mobile Responsive** — Sheet drawer sidebar on mobile, collapsible sidebar on desktop
- **Dark Mode** — Full theme support with OKLch color system

## Getting Started

### Prerequisites

- Node.js >= 20
- A Firebase project with Firestore + Authentication enabled
- A Stripe account (for billing features)
- At least one LLM API key (Anthropic, OpenAI, or Google)

### Installation

```bash
git clone https://github.com/berch-t/kopern
cd kopern
npm install
```

### Environment Variables

Create `.env.local` at the root:

```env
# Firebase Client (public)
NEXT_PUBLIC_FIREBASE_API_KEY=...
NEXT_PUBLIC_FIREBASE_AUTH_DOMAIN=your-project.firebaseapp.com
NEXT_PUBLIC_FIREBASE_PROJECT_ID=your-project
NEXT_PUBLIC_FIREBASE_STORAGE_BUCKET=your-project.firebasestorage.app
NEXT_PUBLIC_FIREBASE_MESSAGING_SENDER_ID=...
NEXT_PUBLIC_FIREBASE_APP_ID=...

# Firebase Admin (server-side only)
FIREBASE_PROJECT_ID=your-project
FIREBASE_CLIENT_EMAIL=firebase-adminsdk-...@your-project.iam.gserviceaccount.com
FIREBASE_PRIVATE_KEY="-----BEGIN PRIVATE KEY-----\n...\n-----END PRIVATE KEY-----\n"

# Stripe
STRIPE_SECRET_KEY=sk_...
STRIPE_WEBHOOK_SECRET=whsec_...
NEXT_PUBLIC_STRIPE_PUBLISHABLE_KEY=pk_...
STRIPE_PRICE_PRO_MONTHLY=price_...
STRIPE_PRICE_PRO_ANNUAL=price_...
STRIPE_PRICE_ENTERPRISE_MONTHLY=price_...
STRIPE_PRICE_ENTERPRISE_ANNUAL=price_...
STRIPE_PRICE_USAGE_INPUT=price_...
STRIPE_PRICE_USAGE_OUTPUT=price_...
STRIPE_PRICE_USAGE_GRADING=price_...
STRIPE_PRICE_USAGE_AUTORESEARCH=price_...

# LLM API Keys (add the ones you need)
ANTHROPIC_API_KEY=sk-ant-...
OPENAI_API_KEY=sk-...
GOOGLE_AI_API_KEY=AI...
OLLAMA_BASE_URL=http://localhost:11434

# Admin (optional)
NEXT_PUBLIC_ADMIN_UID=your-firebase-uid
BUG_FIXER_OWNER_ID=your-firebase-uid
BUG_FIXER_AGENT_ID=               # Fill after creating the Bug Fixer agent in Kopern

# Email notifications (optional)
GMAIL_USER=your.email@gmail.com
GMAIL_APP_PASSWORD=xxxx xxxx xxxx xxxx

# Site URL
NEXT_PUBLIC_SITE_URL=https://kopern.vercel.app
```

### Deploy Firestore Rules

```bash
firebase deploy --only firestore:rules,firestore:indexes
```

### Run

```bash
npm run dev      # Development server on http://localhost:3000
npm run build    # Production build
npm start        # Start production server
npm run lint     # ESLint
npx tsc --noEmit # Type check
```

## MCP Integration

Kopern agents can be used as MCP servers by any compatible client (Claude Code, Cursor, etc.).

### Setup

1. Create an agent in Kopern
2. Go to **MCP Servers** > **New Server** — copy the API key (shown once)
3. Add to your client's `.mcp.json`:

```json
{
  "mcpServers": {
    "my-kopern-agent": {
      "type": "http",
      "url": "https://kopern.vercel.app/api/mcp/server",
      "headers": {
        "Authorization": "Bearer kpn_your_api_key_here"
      }
    }
  }
}
```

### Available MCP Tools

| Tool | Description |
|------|-------------|
| `kopern_chat` | Send a message to the agent (with optional conversation history) |
| `kopern_agent_info` | Get agent metadata (name, description, model, config) |

### Legacy JSON-RPC

The original endpoint is still available for backward compatibility:

```bash
curl -X POST https://kopern.vercel.app/api/mcp \
  -H "Authorization: Bearer kpn_your_api_key" \
  -H "Content-Type: application/json" \
  -d '{"jsonrpc":"2.0","method":"completion/create","params":{"message":"Hello"},"id":1}'
```

### Security Model

- **Firestore Rules** — Owner-only access (`request.auth.uid == userId`), usage docs read-only for client, `apiKeys` admin-only
- **API Key Auth** — SHA-256 hashed storage, prefix-only display, server-side generation
- **Plan Enforcement** — `checkPlanLimits()` on all API routes (12 check types: tokens, agents, grading, teams, pipelines, mcp, meta_agent, sub_agents, autoresearch, github, version_history, model)
- **Model Restriction** — Starter limited to Sonnet + Haiku to prevent cost abuse
- **Stripe Security** — Webhook signature verification, Stripe IDs stripped from client responses, transactional customer creation
- **Admin Features** — Gated via `NEXT_PUBLIC_ADMIN_UID` env var (comma-separated UIDs)
- **GitHub Auth** — Direct popup + credential linking (no `fetchSignInMethodsForEmail`)

## Deployment

### Vercel (Recommended)

1. Push to GitHub
2. Import in Vercel
3. Add all `.env.local` variables in Vercel dashboard
4. Deploy

### Stripe Setup

1. Create products/prices in Stripe Dashboard (Pro monthly/annual, Enterprise monthly/annual)
2. Create Billing Meters for usage-based tracking (input tokens, output tokens, grading runs, autoresearch)
3. Configure webhook endpoint: `https://your-domain.com/api/stripe/webhook`

## Contributing

Kopern is open source and we welcome contributions! Whether it's a bug fix, a new feature, better docs, or a typo — every PR counts.

### How to contribute

1. **Fork** the repo and clone your fork
2. Create a feature branch: `git checkout -b feat/your-feature`
3. Install dependencies: `npm install`
4. Make your changes — run `npm run dev` to test locally
5. Ensure quality:
   ```bash
   npm run lint       # ESLint
   npx tsc --noEmit   # Type check
   npm run build      # Production build passes
   ```
6. Commit with a clear message following [Conventional Commits](https://www.conventionalcommits.org/)
7. Push and open a **Pull Request** against `main`

### What we're looking for

- **New agent templates** for the Examples gallery
- **Grading criteria** — new criterion types or improvements
- **LLM providers** — add support for new providers in `lib/llm/client.ts`
- **Tool integrations** — pre-built tools for popular services (Notion, Linear, Discord...)
- **i18n** — translations beyond EN/FR
- **Performance** — bundle size, Firestore query optimizations
- **Tests** — unit, integration, E2E
- **Documentation** — tutorials, examples, guides

### Guidelines

- TypeScript strict mode — no `any` types
- Use shadcn/ui primitives for all new UI
- Icons from lucide-react only
- Follow existing patterns: `useAuth()`, `useDocument()`, `useCollection()`, `LocalizedLink`, `useDictionary()`
- All new pages must support both EN and FR
- Keep PRs focused — one feature or fix per PR

### Reporting bugs

Use the **built-in bug reporter** in the app (navbar), or open an issue on [GitHub Issues](https://github.com/berch-t/kopern/issues) with:
- Steps to reproduce
- Expected vs actual behavior
- Browser / OS / Node version
- Screenshots if relevant
